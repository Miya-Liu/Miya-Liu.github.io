Note: in the following discussion, we assume the target in a optimisation problem is to fine the minimum values of a function with multi-variables: \(f(\theta_{1}, \theta_{2}, \cdots, \theta_{k})\). $$\begin{equation} \begin{gathered} \operatorname*{min} f(\theta_{1}, \theta_{2}, \cdots, \theta_{k}) \end{gathered} \end{equation}$$

  • Gradient Descent

    Image stand on the graph of the target function. From the standing point, if we walk towards the direction of steepest descent at each step, we may end up at the minimum point on the graph.

    Steepest descent is the tangent line of the graph at a point, which is calculated by the derivatives of \(f(\theta_{1}, \theta_{2}, \cdots, \theta_{k})\). Considering the multiple variables, the derivatives are the partial derivative with respect to each of the variables: $$\begin{equation} \begin{gathered} \frac{\partial }{\partial \theta_{i}} f(\theta_{1}, \theta_{2}, \cdots, \theta_{k}) \end{gathered} \end{equation}$$

    Length of step is called step size or learning rate (in ML), \(\alpha\). If \(\alpha\) is too small, gradient descent can be slow. On the other hand, if \(\alpha\) is too large, gradient descent can fail to overshoot the minimum. (Please refer to the contour line examples in the course of Machine Learning by Andrew Ng.)

    Pseudocode

    repeat until convergence {
         \(\theta_{i} := \theta_{i} - \alpha \frac{\partial }{\partial \theta_{i}} f(\theta_{1}, \theta_{2}, \cdots, \theta_{k})\), (simultaneously update for \(i=1, 2, \cdots, k.\))
    }

    There are two ways to adopt gradient descent: batch gradient descent and stochastic gradient descent. Batch gradient descent uses all the training examples in each step of gradient section, which causes long-time training when using large scale examples, but can guarantee reaching local optimum. Stochastic gradient descent randomly selects and uses only one sample at a gradient descent step. Thus, there is only a selected subset of examples will be used in training, which can speed up the process but may miss the optimum.
